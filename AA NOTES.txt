🔹 EXPERIMENT NO. 1
Implement Symbol Table using Different Data Structures
________________________________________
🎯 1. Objective
To understand:
•	What is a Symbol Table
•	How compiler stores identifiers
•	Operations like insert() and lookup()
________________________________________
📘 2. Theory (Simple Explanation)
A Symbol Table is a data structure used by a compiler to store information about:
•	Variables
•	Functions
•	Constants
•	Identifiers
It stores details like:
Name	Type	Scope	Value
________________________________________
🔧 Operations on Symbol Table
1.	Insert() → Add new identifier
2.	Lookup() → Search identifier
3.	Display() → Show symbol table
________________________________________
🗂 Data Structures Used
1.	Linear List (Array) ✅ (Easy – mostly used in practical)
2.	Hash Table
3.	Binary Search Tree
👉 Here we implement using Linear List (Array).
________________________________________
💻 3. C Program – Symbol Table using Array
#include <stdio.h>
#include <string.h>

#define MAX 100

// Structure of Symbol Table
struct symbol
{
    char name[50];
    char type[20];
    char scope[20];
};

struct symbol table[MAX];
int count = 0;

// Insert function
void insert()
{
    printf("Enter Identifier Name: ");
    scanf("%s", table[count].name);

    printf("Enter Data Type: ");
    scanf("%s", table[count].type);

    printf("Enter Scope (Global/Local): ");
    scanf("%s", table[count].scope);

    count++;
    printf("Identifier Inserted Successfully!\n");
}

// Lookup function
void lookup()
{
    char search[50];
    int i, found = 0;

    printf("Enter Identifier to Search: ");
    scanf("%s", search);

    for(i = 0; i < count; i++)
    {
        if(strcmp(search, table[i].name) == 0)
        {
            printf("Identifier Found!\n");
            printf("Name: %s\nType: %s\nScope: %s\n",
                   table[i].name, table[i].type, table[i].scope);
            found = 1;
            break;
        }
    }

    if(found == 0)
        printf("Identifier Not Found!\n");
}

// Display function
void display()
{
    int i;

    printf("\n----- SYMBOL TABLE -----\n");
    printf("Name\tType\tScope\n");

    for(i = 0; i < count; i++)
    {
        printf("%s\t%s\t%s\n",
               table[i].name,
               table[i].type,
               table[i].scope);
    }
}

int main()
{
    int choice;

    while(1)
    {
        printf("\n1. Insert\n2. Lookup\n3. Display\n4. Exit\n");
        printf("Enter Choice: ");
        scanf("%d", &choice);

        switch(choice)
        {
            case 1: insert();
                    break;

            case 2: lookup();
                    break;

            case 3: display();
                    break;

            case 4: return 0;

            default: printf("Invalid Choice!\n");
        }
    }
}

🔹 EXPERIMENT NO. 4
Construct a Recursive Descent Parser for Arithmetic Expressions
________________________________________
🎯 1. Objective
•	To understand Recursive Descent Parsing
•	To implement LL(1) grammar
•	To check whether an arithmetic expression is valid or invalid
________________________________________
📘 2. Theory (Simple Explanation)
🔹 What is Recursive Descent Parser?
It is a top-down parser.
•	Uses recursive functions
•	One function for each non-terminal
•	Starts from start symbol
•	Checks input according to grammar rules
________________________________________
🔹 Grammar Used (LL(1))
E  → T E'
E' → + T E' | - T E' | ε
T  → F T'
T' → * F T' | / F T' | ε
F  → (E) | id
Where:
•	E = Expression
•	T = Term
•	F = Factor
•	id = identifier (like a, b, x)
________________________________________
💻 3. C Program – Recursive Descent Parser
Save as: rdp.c
#include <stdio.h>
#include <string.h>
#include <ctype.h>

char input[100];
int i = 0;
int error = 0;

void E();
void Eprime();
void T();
void Tprime();
void F();

// E → T E'
void E() {
    T();
    Eprime();
}

// E' → + T E' | - T E' | ε
void Eprime() {
    if(input[i] == '+' || input[i] == '-') {
        i++;
        T();
        Eprime();
    }
}

// T → F T'
void T() {
    F();
    Tprime();
}

// T' → * F T' | / F T' | ε
void Tprime() {
    if(input[i] == '*' || input[i] == '/') {
        i++;
        F();
        Tprime();
    }
}

// F → (E) | id
void F() {
    if(input[i] == '(') {
        i++;
        E();
        if(input[i] == ')')
            i++;
        else
            error = 1;
    }
    else if(isalnum(input[i])) {
        i++;
    }
    else {
        error = 1;
    }
}

int main() {
    printf("Enter Arithmetic Expression: ");
    scanf("%s", input);

    E();

    if(input[i] == '\0' && error == 0)
        printf("Valid Expression\n");
    else
        printf("Invalid Expression\n");

    return 0;
}

🧠 How It Works
Function	Meaning
E()	Expression
T()	Term
F()	Factor
i	Pointer to current input character
error	Checks syntax error
Parser checks expression according to grammar rules step-by-step.
________________________________________
🎓 Viva Questions
❓ What is Recursive Descent Parser?
A top-down parser using recursive functions.
❓ Why remove left recursion?
Because recursive descent parser cannot handle left recursion.
❓ What is LL(1)?
Left-to-right scan, Leftmost derivation, 1 lookahead symbol.
❓ Difference between Top-down and Bottom-up?
Top-down → Starts from start symbol
Bottom-up → Starts from input string

In this experiment, we implemented a recursive descent parser that checks whether an arithmetic expression is syntactically correct.

💻 4. YACC Program (type.y)
Save as: type.y
%{
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

void yyerror(const char *s);
int yylex();

// Simple symbol table
char var[10][10];
char type[10][10];
int count = 0;

char* getType(char *v) {
    for(int i = 0; i < count; i++) {
        if(strcmp(var[i], v) == 0)
            return type[i];
    }
    return "undefined";
}
%}

%union {
    char* str;
}

%token <str> ID
%token INT FLOAT NUM FNUM

%%
program:
      program stmt
    | stmt
    ;

stmt:
      INT ID '=' NUM ';'
        {
            strcpy(var[count], $2);
            strcpy(type[count], "int");
            count++;
            printf("Valid Declaration\n");
        }

    | FLOAT ID '=' FNUM ';'
        {
            strcpy(var[count], $2);
            strcpy(type[count], "float");
            count++;
            printf("Valid Declaration\n");
        }

    | ID '=' ID ';'
        {
            char *t1 = getType($1);
            char *t2 = getType($3);

            if(strcmp(t1, t2) == 0)
                printf("Valid Assignment\n");
            else
                printf("Type Mismatch Error\n");
        }
    ;
%%

int main() {
    printf("Enter Statements:\n");
    yyparse();
    return 0;
}

void yyerror(const char *s) {
    printf("Syntax Error\n");
}

🔹 What is Intermediate Code?
•	It is machine-independent code 
•	Generated between: 
o	Source Code → Machine Code 
________________________________________
🔹 What is Three Address Code (TAC)?
•	Each instruction has at most 3 operands 
👉 Format:
result = operand1 op operand2
________________________________________
✅ Example
Input Expression:
a = b + c * d
Output (TAC):
t1 = c * d
t2 = b + t1
a = t2
________________________________________
💻 3. C Program – TAC Generator
This program generates TAC for simple expressions.
#include <stdio.h>
#include <string.h>

char input[20];
int i = 0;

void generateTAC() {
    char op, op1, op2;
    char temp = 't';

    while(input[i] != '\0') {
        if(input[i] == '+' || input[i] == '-' || 
           input[i] == '*' || input[i] == '/') {

            op = input[i];
            op1 = input[i-1];
            op2 = input[i+1];

            printf("%c = %c %c %c\n", temp, op1, op, op2);

            input[i+1] = temp;  // replace with temp variable
            temp++;
        }
        i++;
    }
}

int main() {
    printf("Enter Expression: ");
    scanf("%s", input);

    generateTAC();

    return 0;
}

⚠️ Note
👉 This program works for:
•	Simple expressions 
•	Single character operands 
(Not for complex nested expressions — but enough for viva/practical)

🔹 EXPERIMENT NO. 9
Code Optimization (Basic Block Optimization)
________________________________________
🎯 1. Objective
•	To perform code optimization 
•	To eliminate redundant expressions 
•	To improve efficiency of intermediate code 
________________________________________
📘 2. Theory (Simple Explanation)
🔹 What is Code Optimization?
It is the process of:
•	Improving code efficiency 
•	Reducing execution time 
•	Minimizing redundant computations 
________________________________________
🔹 What is a Basic Block?
A Basic Block is:
•	A sequence of instructions 
•	With single entry and single exit 
•	No branching inside 
________________________________________
🔹 Example (Unoptimized Code)
t1 = a + b
t2 = a + b
t3 = t1 * c
👉 Here, a + b is calculated twice ❌
________________________________________
✅ Optimized Code
t1 = a + b
t3 = t1 * c
👉 Removed redundant computation ✔️
________________________________________
🔹 Types of Optimization
•	Common Subexpression Elimination ✅ 
•	Dead Code Elimination 
•	Constant Folding 
•	Strength Reduction 
________________________________________
💻 3. C Program – Simple Optimization
This program removes duplicate expressions.
#include <stdio.h>
#include <string.h>

struct expr {
    char op1, op2, op;
} e[10];

int main() {
    int n, i, j;
    
    printf("Enter number of expressions: ");
    scanf("%d", &n);

    for(i = 0; i < n; i++) {
        printf("Enter expression %d (format: a + b): ", i+1);
        scanf(" %c %c %c", &e[i].op1, &e[i].op, &e[i].op2);
    }

    printf("\nOptimized Expressions:\n");

    for(i = 0; i < n; i++) {
        int flag = 0;

        for(j = 0; j < i; j++) {
            if(e[i].op1 == e[j].op1 &&
               e[i].op2 == e[j].op2 &&
               e[i].op == e[j].op) {
                flag = 1;
                break;
            }
        }

        if(flag == 0) {
            printf("t%d = %c %c %c\n", i+1, e[i].op1, e[i].op, e[i].op2);
        }
    }

    return 0;
}

🔹 EXPERIMENT NO. 10
Code Generation (Simple Target Code Generation)
________________________________________
🎯 1. Objective
•	To generate target code 
•	To convert intermediate code (TAC) into assembly-like code 
•	To understand final phase of compiler 
________________________________________
📘 2. Theory (Simple Explanation)
🔹 What is Code Generation?
•	Final phase of compiler 
•	Converts intermediate code → machine/assembly code 
________________________________________
🔹 Input (Three Address Code)
t1 = a + b
t2 = t1 * c
________________________________________
🔹 Output (Target Code)
MOV a, R0
ADD b, R0
MOV R0, t1

MOV t1, R0
MUL c, R0
MOV R0, t2
________________________________________
🔹 Registers Used
•	R0 → temporary register 
________________________________________
💻 3. C Program – Simple Code Generator
#include <stdio.h>

int main() {
    char op, op1, op2, res;

    printf("Enter expression (format: a+b): ");
    scanf("%c%c%c%c", &res, &op1, &op, &op2);

    switch(op) {
        case '+':
            printf("MOV %c, R0\n", op1);
            printf("ADD %c, R0\n", op2);
            printf("MOV R0, %c\n", res);
            break;

        case '-':
            printf("MOV %c, R0\n", op1);
            printf("SUB %c, R0\n", op2);
            printf("MOV R0, %c\n", res);
            break;

        case '*':
            printf("MOV %c, R0\n", op1);
            printf("MUL %c, R0\n", op2);
            printf("MOV R0, %c\n", res);
            break;

        case '/':
            printf("MOV %c, R0\n", op1);
            printf("DIV %c, R0\n", op2);
            printf("MOV R0, %c\n", res);
            break;

        default:
            printf("Invalid Operator\n");
    }

    return 0;
}
